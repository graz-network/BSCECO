# Reuse guide

## Goal

Make the project understandable and reusable by students.

## Reuse rules

1. Keep a clean stage-by-stage audit trail.
2. Preserve the separation between source summary, expert critique, and final synthesis.
3. State clearly what was inherited from the original thesis.
4. State clearly what was updated.
5. State clearly what is newly contributed.
6. Keep a final list of open questions for later cohorts.

## Suggested folder logic for student projects

- inputs/
- notes/
- reviews/
- article/
- exports/

## Minimum documentation at each stage

- files used
- date of the pass
- task performed
- result obtained
- unresolved issue

## Pedagogical advice

Students should first produce a strong article blueprint before trying to write a full article.
